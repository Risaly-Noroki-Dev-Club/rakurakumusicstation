# 热重载播放列表功能测试

## 功能概述
Rakuraku Music Station 现在支持热重载播放列表功能。上传音乐文件后，文件将立即添加到播放列表中，无需重启服务器。

## 实现改进

### 1. 文件保存路径修正
- **问题**: 之前上传的文件保存到当前目录而不是media目录
- **修复**: 现在文件正确保存到`./media/`目录
- **代码位置**: `handle_upload` 函数，第1117行

### 2. 播放列表同步
- **功能**: 上传成功后自动添加到播放列表
- **触发**: 文件保存后立即将文件名添加到播放列表中
- **代码位置**: `handle_upload` 函数，第1126-1127行

### 3. 权限简化（临时）
- **修改**: 上传接口临时启用认证以便测试
- **目标**: 未来将集成完整的认证系统
- **代码位置**: `/upload` 路由，第952行

## 测试步骤

### 1. 启动服务器
```bash
cd dist
./start.sh
```

### 2. 上传音乐文件
使用Web界面或API上传音乐文件到：
- URL: `http://localhost:2240/upload`
- 方法: POST
- 支持格式: MP3, WAV, FLAC, OGG, M4A, AAC

### 3. 验证热重载
1. 上传文件后，文件立即出现在播放列表中
2. 不需要重启服务器
3. 可以在Web管理界面中看到新的文件

### 4. 验证播放
1. 如果播放列表之前为空，新上传的文件会自动开始播放
2. 可以通过Web界面控制播放、暂停、切换曲目

## API测试示例

### 上传文件 (curl)
```bash
curl -X POST http://localhost:2240/upload \
  -F "file=@/path/to/your/song.mp3"
```

### 获取播放列表
```bash
curl http://localhost:2240/api/playlist
```

### 播放控制
- 下一首: `POST /api/next`
- 上一首: `POST /api/prev` 
- 播放指定: `POST /api/play/<index>`

## 文件结构验证
上传的文件会保存到：
```
dist/media/                # 音频文件目录
  ├── song1.mp3           # 上传的文件
  ├── song2.wav
  └── ...
```

播放列表中的文件名为相对路径（仅文件名），播放时会自动加上`./media/`前缀。
